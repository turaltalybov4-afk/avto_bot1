# Боевой чеклист администратора (15 шагов)

Используй этот файл каждый раз, когда подключаешь нового клиента.
Команды бери из файла COMMANDS_COPYPASTE.txt по указанным номерам.

1. Согласовать с клиентом базовые данные:
- Название сервиса
- Рабочие часы
- 8 основных услуг и цены
- Доп. услуги (если есть)
- Telegram ID менеджера

2. Создать отдельный профиль клиента в папке profiles:
- Скопировать profiles/template_client.py
- Назвать файл безопасно: client_slug.py (без пробелов)

3. Внести в профиль клиента:
- TOKEN
- MANAGERS
- COMPANY_NAME
- BUSINESS_HOURS
- SERVICES
- UPSELL

4. Назначить отдельную базу данных клиенту:
- DATABASE_FILE = auto_<client_slug>.db

5. Локально проверить, что профиль запускается без ошибок:
- В терминале выбрать AUTOBOT_PROFILE = client_slug
- Запустить main.py

6. Проверить в Telegram, что бот отвечает на /start.

7. Подключиться к VPS (команда №1).

8. Открыть env файл на сервере (команда №2).

9. В env указать данные клиента:
- BOT_TOKEN=...
- AUTOBOT_PROFILE=client_slug
- AUTOBOT_MANAGERS=...
- AUTOBOT_DATABASE_FILE=/var/lib/auto_bot/auto_<client_slug>.db

10. Запустить сервис (команда №3).

11. Проверить статус сервиса (команда №4):
- Должно быть active (running)

12. Включить автозапуск после перезагрузки сервера (команда №5).

13. Открыть живые логи и проверить, что нет ошибок (команда №6).

14. Сделать тестовую заявку в Telegram:
- Услуга выбирается
- Время выбирается
- Заявка доходит менеджеру
- Заявка сохраняется

15. Передать клиенту мини-инструкцию:
- Как проверить, что бот жив (команда №4)
- Как смотреть логи (команда №6)
- Как перезапустить бота (команда №7)

---

## Быстрая диагностика
- Не запускается: команда №9 (последние 100 строк логов)
- После правок не обновилось: команда №7 (перезапуск)
- Нужно остановить бота: команда №8
- Обновить код с GitHub: команда №10
